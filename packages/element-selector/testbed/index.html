@amplitude/element-selector

—

Interactive testbed running the real package. Click any element in a scenario card to see what selector the engine emits. Toggle the enabled flag in the editor on the right to see the kill-switch behavior — when off, every selector flips to the legacy cssPath algorithm autocapture uses today.

Scenarios

Each card renders one fixture from test/scenarios/. The highlighted element is the canonical target for that scenario, but you can click any element to see the live selector. Round-trip status flashes green if querySelector resolves the emitted selector back to the clicked element.

The Run sessions button on each card re-renders the scenario 10 times with autogenerated ids regenerated between each render — simulating what frameworks do on remount. The result is a stability score for v1 vs. legacy cssPath: 1 of 10 means the algorithm produced the same selector every session; 10 of 10 means a different one each time. The v1 design target is 1 of 10 on every scenario.

Free-form playground

Paste any HTML, then click any element inside the rendered area to see the engine's output. Useful for ad-hoc repros of customer DOMs.
