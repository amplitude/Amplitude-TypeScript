Long Task Test

Open the Network tab or Amplitude debugger to see [Amplitude] Long Task events fire. Each test below is designed to trigger a specific scenario.

1. Basic long task (no measure)

Blocks the main thread for 200ms. Expect: event with no [Amplitude] Long Task Measures.

2. Long task with measure correlation

Wraps the busy work in a performance.measure(). Expect: event with [Amplitude] Long Task Measures: ["heavy-calculation"].

3. Long task after a click

Simulates a click handler that triggers a long task. Useful for correlating tasks with user actions in Amplitude.

4. Multiple overlapping measures

Two measures overlap the same long task. Expect: [Amplitude] Long Task Measures: ["step-one", "step-two"].

5. Short task (should NOT fire)

Blocks for only 10ms — below the 50ms threshold. Expect: no event.
